## What is CodeMyPage?

CodeMyPage is a **drag-and-drop web page editor** designed for educators and content creators. It allows you to build interactive educational pages without any programming knowledge.

### What You Can Create

- 📝 **Text & Media** - Headlines, paragraphs, images, videos, audio
- 📋 **Interactive Elements** - Quizzes, fill-in-the-blank, multiple choice, true/false
- 🔢 **Sorting & Ordering** - Drag-and-drop sorting exercises
- 🎨 **Creative Tools** - Drawing boards for freehand artwork
- 📊 **Learning Aids** - Progress bars, timelines, checklists
- ✅ **Feedback Components** - Answer sheets, score displays, explanations

## ⚠️ Before You Export

### What the Export Does

When you click **Export**, the editor creates a file that can run on any computer. This is powerful, but it also means you need to be careful.

**Here's what happens:**

- **The exported file is a small program** - It contains all your content plus the tools to display it
- **It needs to download some code** - The file will automatically download React (a code library) from the internet when it opens
- **Your content is inside the file** - Everything you created is stored in the exported file

**How to stay safe:**

- 🔒 Only use this editor from **websites you trust** (like the official GitHub Pages)
- 🔒 **Never open exported files from unknown sources** - They could contain harmful code
- 🔒 If you get a file from someone else, **scan it with your antivirus first**
- 🔒 Download the editor directly from **https://** websites (not http://)

**Red flags to watch for:**

- Someone sends you an exported file unexpectedly
- The file is from a website you've never heard of
- Your antivirus warns you about the file

### Export Technical Details

#### Export Process

When the Interactive Export feature is triggered, the editor performs the following operations:

1. **Build Phase**: The server-side Vite build system compiles the complete React application containing all project components, styles, and configurations into optimized JavaScript bundles
2. **Package Phase**: The compiled JavaScript and CSS assets are embedded directly into a self-contained HTML document
3. **Runtime Phase**: The exported HTML file loads React and ReactDOM libraries from CDN (unpkg.com) and initializes the application client-side

#### Potential Security Concerns

| Risk Category | Description | Impact |
|---------------|-------------|--------|
| **Code Injection** | If the editor's source code is maliciously modified, exported files may contain embedded malicious code | High - Full system access potential |
| **Supply Chain Attack** | Exported files depend on external CDN for React/ReactDOM; compromise of CDN infrastructure could lead to malicious code injection | High - Affects all exported files |
| **Data Exposure** | Exported files contain complete project data including all components and configurations | Medium - Intellectual property disclosure |
| **Sandbox Escape** | While the exported file runs in browser sandbox, combined with other vulnerabilities it could potentially exploit browser flaws | Medium - Depends on browser security |

#### Security Recommendations

- Verify source code integrity via git commit signatures or checksum verification
- Use Subresource Integrity (SRI) hashes for CDN dependencies in production deployments
- Implement Content Security Policy (CSP) headers when hosting exported files
- Consider self-hosting React dependencies for sensitive deployments
- Audit exported files with static analysis tools before distribution

#### Technical Specifications

- **Build System**: Vite with custom export configuration
- **Output Format**: Single self-contained HTML file
- **Runtime Dependencies**: React 18.x, ReactDOM 18.x (loaded from CDN)
- **Browser Compatibility**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
