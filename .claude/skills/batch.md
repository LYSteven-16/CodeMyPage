# Batch 技能 — 批量任务处理

当用户说"批量""同时做""一起改""很多地方要改"时触发。
适用于多个独立任务需要同时处理的情况。

## 执行步骤

### 第一步：拆分任务

把用户的任务拆成独立的子任务，每个子任务应该：
- 可以独立完成
- 不依赖其他子任务的输出
- 有明确的验收标准

示例拆解：
```
用户需求：在 10 个文件中添加错误处理

拆解为 10 个子任务：
1. 给 src/a.ts 添加 try-catch
2. 给 src/b.ts 添加 try-catch
...
```

### 第二步：制定执行计划

```
## Batch 执行计划

共 N 个子任务，预计并行执行：

| # | 文件 | 改动内容 | 优先级 |
|---|------|---------|--------|
| 1 | src/a.ts | 添加错误处理 | 高 |
| 2 | src/b.ts | 添加错误处理 | 高 |
|...| ... | ... | ... |

执行顺序：
- 第一批（可并行）：任务 1, 2, 3, 4, 5
- 第二批（可并行）：任务 6, 7, 8, 9, 10
```

### 第三步：执行

按批次执行，每批内部并行处理。

### 第四步：汇总结果

```
## Batch 执行报告

### 完成情况
✅ 任务 1：src/a.ts - 完成
✅ 任务 2：src/b.ts - 完成
❌ 任务 3：src/c.ts - 失败（原因）

### 总结
完成 9/10，失败 1 个需要单独处理
```

## 注意事项
- 每个子任务完成后说明结果
- 失败的任务单独列出原因
- 不确定时先问用户确认再执行
